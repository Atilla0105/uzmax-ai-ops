# 项目输入开工准备清单

来源对齐：`UZMAX智能运营系统-技术架构-v1.1.md` 的 Owner 侧关键路径，以及 `UZMAX智能运营系统-1.0验收矩阵-v1.1.md` 的证据产生者与发布判定规则。

原则：Owner 输入不是上线前补材料，而是里程碑闸门条件。任一输入延期，对应闸门顺延，并在 ADR 或里程碑记录中写明顺延决定、影响范围和恢复日期。

## 提交规则

- 所有文件先进入临时交付区，验收通过后再归档到对应证据位置。
- 含客户、员工、电话、地址、订单号、金额、物流单号、聊天账号、截图头像的信息必须先脱敏。
- 每个交付物必须有项目 owner 确认、提交日期、版本号、覆盖范围、已知缺口和签收状态。
- 不接受只给口头描述；素材、样本、盲评和签收都必须能被复核。
- 每个里程碑关闭时归档本阶段证据，不得堆到 M6 集中补签。
