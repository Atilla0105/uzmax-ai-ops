- Every page worker must inspect the exact target `/Users/atilla/源码/unpacked 6/pages/**` files, source components and relevant owner HTML region before implementation and before any visual acceptance claim. The source already contains layout/component details; workers must not invent layouts, freely rearrange UI or carry old shell visuals.

- Mobile remains an acceptable/readable/no-overflow fallback in this phase. Pixel-level mobile redesign/polish is deferred to a later mobile-specific pass and must not displace desktop parity work.
- Group and tenant layers remain separate: `/design` or admin/home opens group layer/group overview; selecting a tenant enters tenant layer. Later page plans must keep this visible.
- Release/acceptance remains transitional/low business value and must not displace high-value real admin pages.
- Every page PR/evidence must include the exact owner HTML region or screenshot compared, the exact unpacked source files/components inspected, sidebar parity result when visible, mobile fallback result and a remaining desktop visual-delta list. Unresolved visual deltas block visual acceptance unless the owner explicitly accepts them or a documented runtime/permission/degraded state requires the difference.
