# UZMAX 智能运营系统 后台设计与前端架构 v1.1

> 状态：1.0 投入版后台规范修订版  
> 日期：2026-06-13  
> 派生自：`UZMAX智能运营系统-PRD-v1.1.md`  
> 本文件定义“后台长什么样、怎么操作、前端如何保持稳定与一致”。

---

## 1. 设计立场

后台是 **UZMAX 运营塔台**。它不是营销页，也不是轻量聊天插件；它是集团 owner、租户负责人、客服、知识维护者、AI 运营人员每天处理风险、接管会话、修正知识和复盘成本的主工作区。

### 1.1 设计原则

- **状态优先**：颜色、角标、排序、密度都服务于状态判断；红色只表达“需要人工或阻断风险”。
- **管理闭环优先**：任何高风险动作必须能看到上下文、执行动作、留下审计、可回滚或可追踪。
- **高密度但不拥挤**：页面为长时间工作设计，信息表格化、分组清晰、避免大留白和装饰图。
- **键盘优先**：对话切换、接管、工单认领、确认队列、草稿发送、评测失败定位都有快捷键。
- **桌面为主，移动兜底**：完整后台优先桌面；手机只保证告警、审批、工单认领、AI 急停/恢复。

## 2. 信息架构

后台分为 **集团层** 与 **租户层**。

```text
登录
  -> 集团总览
      - 租户健康
      - 模型/成本/风险
      - 模板中心
      - 连接中心
      - 发布与验收
      - 集团日志
      - 租户管理
  -> 选择租户
      - 对话
      - 工单
      - 客户资产
      - 订单
      - 知识与资源
      - 确认队列
      - 评测中心
      - AI 成员
      - 团队
      - 配置
      - 分析
      - 日志
```

### 2.1 全局框架

- 左侧 64px 图标导航，hover/固定展开显示标签。
- 顶栏包含：当前层级、租户切换器、环境标识、系统心跳、搜索、通知、用户菜单。
- 集团层顶栏显示“集团”；租户层顶栏显示“集团 / 租户名”并允许快速切换授权租户。
- production/staging 环境必须有明显顶栏差异。

### 2.2 租户切换器

- 可搜索租户名称、业务线、状态。
- 显示租户健康灯：健康、降级、需人工、熔断。
- 切换租户时清空租户级缓存，重新拉取权限与 feature flags。

## 3. 集团层页面

### 3.1 集团总览

- 顶部健康条：租户总数、异常租户、AI 熔断、模型故障、订单 connector 故障、红线事件。
- 主表：每个租户一行，列为会话量、待人工、SLA 风险、转人工率、AI 成本、评测状态、订单状态、最后异常。
- 支持点击进入租户上下文；集团层不展示客户明文和会话内容。

### 3.2 模型/成本/风险

- 模型任务矩阵：任务、primary、fallback、成本、延迟、失败率、最近评测。
- 成本趋势：集团总成本、按租户成本、按任务成本。
- 风险队列：红线拦截、误报、熔断、异常成本、AllProvidersDown。

### 3.3 模板中心

- 知识模板、AI 成员模板、配置模板、评测模板四类标签页。
- 模板卡显示：适用业务、版本、最近复制租户、通过评测状态。
- “复制到租户”必须二次确认，并在目标租户生成独立版本。

### 3.4 租户管理

- 创建/停用租户、设置默认语言/时区/渠道能力、选择初始模板。
- 显示租户成员数、AI 成员数、渠道连接数、订单 connector 状态。
- 停用租户必须要求输入原因，停用后保留只读审计与数据导出入口。

### 3.5 连接中心

- 集团层维护连接类型卡片：Telegram Bot、Telegram Business、订单 API、导入兜底。
- 卡片显示适用租户数、健康状态、最近错误、spike 结论、feature flag 状态、添加/管理入口。
- Telegram Business 卡片展示 ADR-B01 结论：全部可行、部分可行、不可行；不可行时入口整体隐藏，卡片只保留集团只读状态。
- 订单 API 卡片展示 ADR-B02 结论；无 API 时租户订单页把导入兜底标记为主路径。
- 租户层只看到已授权且 feature flag 开启的连接类型；连接配置、测试、停用和降级原因写操作日志。

### 3.6 发布与验收

- 面向 owner 的发布控制台，展示 M0-M6、GA-0 与正式发布的闸门状态。
- 每个闸门展示：覆盖验收项、证据产生者、证据链接、owner 签收状态、阻断项、ADR 链接。
- GA-0 checklist 必须逐项勾选，全部满足后才显示“开闸”动作；开闸写操作日志。
- 正式发布页显示全部 P0/P1/P2 状态；P0 未过时发布按钮禁用，P1 未过要求 owner 填风险说明与修复日期。

## 4. 租户层页面

租户层导航按业务闭环排序：对话、工单、客户资产、订单、知识与资源、确认队列、评测中心、AI 成员、团队、配置、分析、日志。营销群发、广告分析、外部应用市场不进入 1.0 导航。

### 4.1 对话工作台

三栏结构：

```text
会话列表       对话线程 / 草稿区             上下文面板
筛选/搜索      消息流 + AI轨迹              客户档案
状态分组       接管/放回/发送               工单/订单/报价
SLA计时        Business草稿确认              快捷动作/备注
```

- “待人工”和“SLA 即将超时”永远置顶。
- Bot 消息显示是否自动发送；Business 草稿显示“待确认”，不能伪装成已发送。
- AI 决策痕迹可展开：意图、工具、知识引用、模型、token、成本、红线检查。
- 人工本人在 Business 内发送的消息进入线程，并显示“人工外部回复”。

### 4.2 工单

- 默认视图：待认领、我的、即将超 SLA、已重开、今晨待跟进。
- 工单详情包含：摘要、建议处理、客户档案、相关订单、报价记录、会话片段、备注、事件时间线。
- 关闭必须选择结果：resolved、转人工渠道、无响应、无效、重复，并填写去向或说明。

### 4.3 客户资产

- 列表筛选：语言、文字系统、旅程阶段、未决问题、拉黑、不可达、有订单、有报价。
- 客户详情：身份、档案、自定义字段、客户标签、历史会话、工单、订单快照、报价记录、双轨引导记录、人工备注。
- 客户资产侧栏包含二级入口：客户列表、会话搜索、客户标签、会话标签、自定义字段。
- 会话搜索支持按客户、会话 ID、关键词、时间、渠道、成员过滤，结果可跳回原会话。
- 身份归并/拆分必须展示影响范围，并写审计。
- 匿名化删除入口仅对授权成员可见。

### 4.4 订单

- 搜索：姓名、电话、Telegram 用户、批次号、订单号、外部 ID。
- 订单详情显示来源、更新时间、过期提示、状态、批次、物流节点、关联客户与会话。
- connector 降级时顶部出现琥珀条，提示当前使用导入快照或缓存快照。
- 若 ADR-B02 判定无可用 API，页面文案改为“订单数据主路径：导入快照”，不再把 API 失败伪装成临时异常。
- 导入页显示任务进度、成功行、失败行、错误下载、回滚入口。

### 4.5 知识与资源

- 教程旅程、事实条目、公共话术、私人话术、素材库、模板来源六个标签页。
- 旅程页显示阶段管线、阶段自动定位率、负反馈最高阶段、素材关联状态。
- 条目页显示命中、负反馈、版本、评测状态、红线标记。
- 话术页支持标题/内容搜索、分类、创建人、创建时间、最近编辑人、导入/导出；公共话术归租户共享，私人话术仅本人可见。
- 素材页显示名称、类型、格式、大小、描述、storageRef、本租户 file_id 缓存状态、关联阶段、未引用素材；支持批量删除和上传。

### 4.6 确认队列

- 单列高密度卡片流，键盘 `J/K` 移动，`A` 通过，`E` 编辑，`D` 丢弃。
- 类型包括知识候选、档案候选、评测候选、冲突候选。
- 冲突候选强制并排 diff，不能跳过；未处理不入库。
- 队列顶部显示今日候选数、每日上限 10、7 日通过率、蒸馏频率、最近降频原因。
- 连续 3 日通过率低于 40% 时显示琥珀提示条，提供“查看原因”和“恢复每日”入口；恢复每日必须二次确认。
- 卡片交互按“owner 每日处理 <= 5 分钟”设计：默认展示判断所需最少字段，详情按需展开。

### 4.7 评测中心

- 首页显示当前 production gate：通过/阻断/运行中。
- 评测集标签：意图、教程、报价、截图、乌语拉丁、乌语西里尔、俄语、红线攻击、红线误报、Business 草稿。
- 评测运行详情显示 case 结果、失败输出、模型、prompt 版本、知识版本、人工盲评状态。
- 发布配置时若 gate 未通过，保存按钮禁用并链接到失败项。

### 4.8 AI 成员

- 列表展示人类成员与 AI 成员分区，也可按成员类型统一筛选；AI 成员卡突出状态、能力开关、今日处理量、成本、负反馈。
- AI 状态：在线、手动离线、熔断离线。熔断恢复必须走独立确认流程。
- 能力开关：教程、截图、报价、订单查询、Business 草稿；关闭单能力只影响对应意图。
- 人设编辑有版本对比、评测 gate、回滚。

### 4.9 团队与配置

- 团队：邀请、停用、Telegram 绑定、presence、通知偏好、语言偏好、角色与逐权限授权、接待上限。
- 成员表列：昵称、邮箱/账号、角色、成员类型、分组、在线状态、接待中会话数、今日累计接待、创建时间、最近登录。
- 角色管理列：角色名称、角色类型、描述、创建时间、操作；权限编辑按对话、客户、订单、知识、AI、配置、日志分组。
- 配置：业务配置、SLA、模板、模型路由、成本护栏、熔断阈值、渠道配置、订单 connector。
- 所有配置保存生成版本；回滚必须二次确认并写审计。

### 4.10 分析与日志

- 分析首页保留固定运营看板：解决率、转人工率、SLA、成本、Top 问题、订单查询、草稿采纳、知识健康、确认队列 7 日通过率、蒸馏频率、真实流量基线。
- 分析表支持添加维度：成员、AI 成员、渠道、意图、时间粒度、订单状态、转人工原因；支持导出。
- 日志分为登录日志、在线日志、操作日志。
- 登录日志列：类型、成员、位置/IP、设备、时间。
- 在线日志列：成员、账号状态、更新方式、更新时间、状态持续时长。
- 操作日志列：操作时间、操作人、操作模块、操作功能、操作对象、操作内容；高风险项可跳转到会话、工单、配置版本、评测运行。

## 5. 移动端兜底

移动端只保证以下流程：

- 查看告警详情。
- 认领/转派紧急工单。
- 处理确认队列的通过/丢弃，编辑可跳桌面。
- AI 成员急停、熔断恢复确认。
- GA-0 或正式发布阻断项的只读查看。
- 查看租户健康摘要。

移动端不承诺完整知识编辑、配置矩阵、评测分析、订单导入、复杂客户归并、自定义字段配置和批量资源管理。

## 7. 前端架构

- `ui-tokens -> primitives -> patterns -> pages` 分层；页面禁止直接拼原生样式。
- TanStack Query 管服务端状态；WebSocket 事件直接 patch cache。
- Zustand 只放 UI 状态：当前租户、侧栏、快捷键、草稿编辑状态。
- 会话列表、消息线程、日志列表、订单列表、客户列表必须虚拟化。
- 图表、评测详情、审计详情懒加载。
- `/design` 活体规范页展示 token、primitive、pattern、状态、核心屏；Playwright 截图回归。

## 8. 性能与质量预算

- 首屏进入租户对话工作台可交互 <= 1.5s（模拟 4G），首屏资源 <= 250KB gzip。
- WS 事件到屏幕呈现 <= 200ms。
- 1 万条会话列表滚动稳定 60fps。
- 接管、认领、确认队列操作本地反馈 <= 100ms。
- 每个核心页面必须有 loading、empty、error、permission denied、degraded 状态。
- 表格型页面默认提供搜索、筛选、分页、列宽稳定、空状态动作；导入/导出和批量操作只在确有业务价值的页面出现。
- `/design` 页与对话、确认队列、发布与验收三类核心屏纳入 Playwright 视觉回归。
