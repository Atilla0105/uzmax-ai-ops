# M7-UI-62B Tenant Entry Visible Proof Evidence

## Summary

This slice adds focused browser proof for the existing visible route loop:

`/design` -> `group.overview` -> click `玉珠跨境美妆` -> `tenant.conversations`.

It proves the owner boundary that group layer and tenant layer are separate: default `/design` opens the group layer, and selecting a tenant enters the tenant layer. It is not a redesign and does not claim conversation runtime closure, group runtime closure, authz/cache-invalidation closure, owner final visual acceptance, production readiness, GA-0 or 1.0 release approval.

- `/design` defaults to group shell and `group.overview`.
- Group nav sections are exactly `总览 / 平台 / 治理`; tenant labels are absent.
- Topbar shows `集团`, `玉珠跨境美妆`, search placeholder, `PRODUCTION`, `68ms` and `韩雪`.

- Tenant nav sections are exactly `运营 / 数据 / 智能 / 管理 / 洞察`; group labels are absent.
- Tenant-layer sidebar, topbar and tenant context are visible after entry.
- The flow does not fall back to a single mixed universal dashboard.

- `320px` mobile fallback keeps `document.body.scrollWidth <= 320`.
